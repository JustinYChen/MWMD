# 情侣问题抽卡网站 · 实现计划

## 一、摘要（Summary）

构建一个面向情侣的、极具创意与交互感的「问题抽卡」网站。核心体验是**塔罗牌阵式抽卡**：用户选择抽牌模式（单抽 / 3 张叙事 / 5 张十字）与筛选条件后，3D 牌面朝下铺开，鼠标悬停牌体抬起，点击翻牌揭示一道中英双语问题，供两人互相提问深聊。

视觉风格为**莫兰迪裸色系 + 玫红/金点缀**的简约甜蜜美学，支持深浅主题切换。题库为**原创中英双语问题**（中文主、英文辅），灵感借鉴 We're Not Really Strangers / Let's Get Deep / BestSelf Intimacy Deck / Better Together / Love Birds 等桌游的「深度分级 + 类别」思路，但不照搬原卡文字以规避版权。

技术栈：React + Vite + Tailwind CSS + Framer Motion + GSAP + react-three-fiber（Three.js），zustand 状态管理，localStorage 持久化，部署到 GitHub Pages 纯静态托管。

### 已确认需求（来自两轮澄清）
| 维度 | 决策 |
|---|---|
| 抽卡方式 | 塔罗牌阵（牌面朝下铺开，悬停抬起，点击翻牌） |

### 关键技术调研结论
1. **react-three-fiber 3D 塔罗牌**：用 drei `RoundedBox`（6 面材质数组，索引 4=牌面、5=牌背）+ `CanvasTexture` 程序化生成牌面/牌背（绘制中文问题、玫红花环牌背），翻牌用 `useFrame` 内 delta-based lerp 插值 `rotation.y` 0→π，配合 `position.y` 抬起与 `position.z` 前推。
2. **GSAP / Framer Motion 分工**：Framer Motion 负责 UI 层（路由转场、stagger、hover/tap、模态、`layoutId` 共享布局）；GSAP + `useGSAP`（@gsap/react）负责复杂时间轴与滚动驱动（英雄区入场、ScrollTrigger 揭示、纪念日彩蛋、3D 牌阵飘落入场）。铁律：同一 DOM 节点不被两库同时控制；R3F 内 3D 动画一律用 `useFrame`+lerp，不用 GSAP 直接操作 Three 对象。
3. **Vite 部署 GitHub Pages**：`base` 路径按仓库名配置；路由用 `createHashRouter`（带 `#`）规避刷新 404；`public/` 资源用相对路径 `./`；GitHub Actions `actions/deploy-pages@v4` 部署。
4. **Web Audio API 音效**：翻牌「啪」声（白噪声+低通+指数衰减）、悬停「叮」声（880Hz 正弦+ADSR）、揭示和弦（C-E-G）；AudioContext 须在用户首次交互后 `resume()`；背景音乐用 CC0 授权 mp3 或自合成 ambient pad。

## 六、状态管理（zustand + persist）

5 个 store，localStorage 键命名空间 `cc:*`，每值带版本号 `{ v: 1, data }` 便于迁移：
- **useDeckStore**：`mode` / `filters`（levels、categories、intimacyMax）/ `drawnIds`（防重复）/ `currentDraw` / `phase`（idle→spreading→flipping→revealed）/ `draw()` / `resetDeck()`。
- **useFavoritesStore**：`items` / `add` / `remove` / `isFavorited` / `clear`。
- **useHistoryStore**：`records`（含 `drawnAt`、`mode`、`favorited`）/ `addRecord` / `clearHistory`，限 200 条滚动淘汰。
- **useProfileStore**：`nameA` / `nameB` / `anniversary` / `setupCompleted`。
- **useSettingsStore**：`theme` / `soundEnabled` / `bgmEnabled` / `volume` / `language`。

数据流：选模式+过滤器→`draw()`（按过滤器筛选、排除 `drawnIds`、Fisher-Yates 洗牌取 N 张）→Canvas 渲染牌阵→点击翻牌→揭示→可收藏/再抽/同步写历史。剩余未抽 < 模式所需时提示重置牌堆。zustand 选择器精准订阅避免 Canvas 全量重渲染。

## 七、核心交互实现要点

1. **3D 牌阵布局**（`three/layouts.ts`）：单抽居中；3 张横排（过去-现在-未来/你-我-我们）；5 张十字（中心+四方位）。入场用 GSAP 时间轴 stagger，牌从 `y:-5` + 随机 `rotation.z` 飘落到目标位。
2. **TarotCard 翻牌与悬停**：`RoundedBox args=[1.4,2.2,0.02] radius={0.04}`，6 面材质数组索引 4=牌面/5=牌背；`useFrame` delta-based lerp（系数 `1-exp(-delta*8)`）插值 `position.y`（hover+0.4）、`rotation.y`（flip 0→π）、`scale`（hover 1.05）；`onPointerOver/Out` 切换 hover+预览音，`onClick` 翻牌+翻牌音+`onReveal` 回调。
3. **CanvasTexture 生成**（`three/textures.ts`）：512×768 canvas，背景按深度取色渐变、金色边框、类别图形（Path 绘制避免图标字体）、中文自动换行（Noto Serif SC）、英文小字（Cormorant Garamond）。**须 `await document.fonts.ready` 后再生成**，否则首张牌渲染默认字体。
4. **GSAP/FM 分工**：路由转场=FM `AnimatePresence mode="wait"`；模式选择=FM `layoutId`；网格 stagger=FM variants；英雄区入场+ScrollTrigger 揭示+纪念日彩蛋+3D 飘落=GSAP `useGSAP({scope})`。
5. **Web Audio 音效**（`lib/audioEngine.ts`）：单例 AudioContext，`playFlip`/`playHover`/`playReveal`/`startBgm`，首次交互后 `init()`+`resume()`。
6. **主题切换**：`data-theme` 属性 + CSS 变量；3D texture 随主题重生（dispose 旧 texture）。
7. **纪念日彩蛋**（`useAnniversary`）：检测今天是否为纪念日或里程碑天数（100/520/1314），触发全屏 GSAP 序列（花瓣飘落+祝福文字+烟花粒子）。
8. **GitHub Pages 部署**：`vite.config.ts` base 按仓库名配置；`createHashRouter`；GitHub Actions `actions/deploy-pages@v4`；`public/` 资源相对路径。

## 九、假设与决策（Assumptions & Decisions）

1. **项目位置**：在 `couple-cards/` 子目录创建独立 Vite 项目，不破坏根目录现有 Goyard 项目。如用户希望直接部署到仓库根 URL，可调整 base 路径或将构建产物提到根。
2. **部署 base 路径**：默认按仓库名 `/<repo-name>/` 配置（生产环境），本地开发 `/`。用户需告知 GitHub 仓库名以最终确定；若为 `user.github.io` 顶级仓库则 base 为 `/`。
3. **题库版权**：全部原创编写，仅借鉴深度分级与类别思路，不引用任何桌游原卡文字。
4. **背景音乐版权**：优先 Web Audio 自合成 ambient pad；若用 mp3 必须 CC0/授权（Pixabay Music、Free PD）。
5. **路由模式**：采用 `createHashRouter`（URL 带 `#`）以最稳地规避 GitHub Pages 刷新 404，无需额外 404.html。
6. **状态持久化**：用 zustand persist + localStorage，无需后端；history 限 200 条滚动淘汰防配额超限。
7. **语言切换**：卡面固定「中文主+英文辅」；全局 UI 文案通过 `lib/i18n.ts` 支持中英切换（默认中文）。
